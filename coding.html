  • The Philosophy of Software Design by John Ousterhout (designer of MongoDB, CockroachDB and Kafka), 2025
    Software designing becomes more important when AI tools come.
    Always try a second idea to finish the task. It usually would be better, or you would learn something.
    Posing more error exceptions may overcomplicate the design. Some errors never exist.
    Never repeat yourself during discussion.
    Get feedback and redo your task. Almost all learnings are about making mistakes.
    Decomposition is the key to CS design.
    Designing (i.e., decomposite to which degree) requires good trade-off.
    Generally, short-line methods are good, but pursing this extrmely makes it hard to understand a function.
    Functions that are closely related should be combined and put together: Do not over-decomposite.
    Writing tests before coding makes the code overfitting special tests, instead of a general solution.
    Comments should never repeat the code. Comments are essential in interfaces and class members.
